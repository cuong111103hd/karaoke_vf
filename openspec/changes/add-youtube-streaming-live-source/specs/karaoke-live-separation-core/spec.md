## MODIFIED Requirements

### Requirement: Live producer creates source chunks
The system SHALL create sequential source audio chunks for live separation as soon as enough source audio is available, without requiring full-song YouTube download and full-file normalization first when streaming source mode is selected.

#### Scenario: Source chunk becomes available
- **WHEN** enough source audio is available for the next chunk
- **THEN** the system writes a deterministic source chunk file and records it in the manifest

#### Scenario: Streaming source chunk becomes available before full download
- **WHEN** streaming source mode has decoded enough audio for the next planned chunk window
- **THEN** the live producer uses the finalized source chunk file for separation without waiting for the complete YouTube media file

## ADDED Requirements

### Requirement: Live producer selects source mode
The system SHALL support a configurable live source mode for source chunk generation.

#### Scenario: Streaming source mode selected
- **WHEN** live options specify streaming source mode
- **THEN** the live producer uses the direct-URL streaming source to generate source chunks

#### Scenario: Download source mode selected
- **WHEN** live options specify download source mode
- **THEN** the live producer uses the existing full-download and normalization source path

### Requirement: Live producer records source timing
The system SHALL record timing metadata for source preparation, streaming startup, source chunk waiting, and first source chunk readiness.

#### Scenario: First streaming source chunk ready
- **WHEN** source chunk 0 is finalized by the streaming source
- **THEN** the manifest timing metadata includes markers that allow first-source-chunk latency to be measured

#### Scenario: Chunk waits for source audio
- **WHEN** the live producer waits for a source chunk to become available
- **THEN** the chunk timing metadata records the source wait duration

### Requirement: Live producer preserves separator and playback contracts
The system SHALL keep downstream separation and playback behavior file-compatible when using streaming source mode.

#### Scenario: Separator receives streaming source chunk
- **WHEN** a source chunk is generated by the streaming source
- **THEN** the selected separator receives a finalized source WAV file path using the existing separator contract

#### Scenario: Playback consumes streaming-derived instrumental chunk
- **WHEN** the selected separator finishes a streaming-derived source chunk
- **THEN** the live manifest publishes the instrumental chunk path using the existing ready-chunk behavior
